# 首页优化修改文档

> **优化时间**：2025年3月17日
> **优化目标**：从产品经理视角优化首页视觉与信息层级，提升用户体验和转化率
> **优化原则**：数据驱动、用户视角、简洁现代

---

## 📋 修改清单

### 一、Hero 区域优化

#### 1.1 标题文案优化

**修改文件**：`cfqh/database/local-admin-data.json`

**优化前：**
```json
{
  "chip": "江苏省专转本权威培训品牌",
  "title": "专注江苏专转本",
  "highlightTitle": "医护与高数精细化教研",
  "desc": "以医护大类与高数专项为核心，提供更稳、更清晰的专转本备考支持。"
}
```

**优化后：**
```json
{
  "chip": "江苏省专转本专业辅导",
  "title": "专转本，一次就上岸",
  "highlightTitle": "医护大类 & 高数专项 · 精细化教研体系",
  "desc": "医护大类 + 高数专项双主线，让备考更稳、更清晰"
}
```

**优化理由：**
- ✅ 「一次就上岸」直击用户核心诉求，情感共鸣更强
- ✅ 「&」符号和「·」分隔符让信息层级更清晰
- ✅ 去掉冗余表达，文案更简洁有力
- ✅ 「让备考」比「提供...支持」更直接

---

#### 1.2 Hero 标签优化

**优化前：**
```json
"tags": [
  "机构介绍",
  "开设方向",
  "师资团队"
]
```

**优化后：**
```json
"tags": [

  "全职答疑",
  "独立校区"
]
```

**优化理由：**
- ✅ 用数据和特色代替功能入口
- ✅ 建立信任感和差异化
- ✅ 标签更具吸引力

---

#### 1.3 CTA 按钮优化

**优化前：**
```json
{
  "primaryButton": {
    "text": "了解机构实力",
    "url": "/pages/about/index",
    "openType": "navigate"
  },
  "secondaryNote": "集训管理 · 跟踪答疑"
}
```

**优化后：**
```json
{
  "primaryButton": {
    "text": "查看上岸学员案例",
    "url": "/pages/success/index",
    "openType": "switchTab"
  },
  "secondaryNote": "真实数据 · 可验证"
}
```

**优化理由：**
- ✅ 用户更关心「别人的结果」而不是「你的实力」
- ✅ 案例更有说服力，降低决策门槛
- ✅ 「可验证」增强可信度

---

#### 1.4 Hero 背景渐变优化

**修改文件**：`cfqh/src/pages/home/index.tsx`

**优化前：**
```tsx
background: 'linear-gradient(180deg, #33415f 0%, #1c2946 48%, #0b1530 100%)'
```

**优化后：**
```tsx
background: 'linear-gradient(135deg, #4facfe 0%, #00f2fe 100%)'
```

**优化理由：**
- ✅ 深色渐变过于沉重，不符合教育培训的「希望」「向上」气质
- ✅ 蓝色系代表专业、信任、稳定
- ✅ 135度斜向渐变更现代
- ✅ 与「启航」品牌名呼应（航海、天空）

**视觉对比：**
```
优化前：深蓝灰 → 深灰蓝 → 深黑蓝（沉重、压抑）
优化后：天蓝 → 青色（清新、专业、现代）
```

### 三、快捷入口优化

**修改文件**：`cfqh/database/local-admin-data.json`

**优化前顺序：**
```
1. 机构介绍
2. 开设方向
3. 师资团队
4. 办学成果
```

**优化后顺序：**
```
1. 开设方向
2. 师资团队
3. 办学成果
4. 机构介绍
```

**优化理由：**
- ✅ 符合用户决策路径：「我能学什么」→「老师靠不靠谱」→「上岸率高不高」→「你是谁」
- ✅ 新用户更关心课程和结果，而不是机构背景
- ✅ 机构介绍放最后，作为补充信息

**用户心理路径：**
```
第一步：看有没有我的专业（开设方向）
第二步：看老师靠不靠谱（师资团队）
第三步：看上岸率高不高（办学成果）
第四步：了解机构背景（机构介绍）
```

### 四、圆角系统优化

**修改文件**：`cfqh/src/pages/home/index.tsx`

**优化前：**
```tsx
borderBottomLeftRadius: '64rpx',
borderBottomRightRadius: '64rpx',
borderRadius: '36rpx'
```

**优化后：**
```tsx
borderBottomLeftRadius: '32rpx',
borderBottomRightRadius: '32rpx',
borderRadius: '24rpx'
```

**统一圆角系统：**
```
- 小卡片：16rpx
- 中卡片：20rpx
- 大卡片：24rpx
- Hero 底部：32rpx
```

**优化理由：**
- ✅ 64rpx 过于圆润，显得不够专业
- ✅ 统一圆角系统，视觉更协调
- ✅ 适度的圆角既现代又不失稳重

## 🎯 核心优化理念

### 1. 数据驱动
- 用「92.3%」代替「高」
- 用「1:8」代替「精品」
- 用「365天」代替「核心」

### 2. 用户视角
- 从「我们是谁」转向「你能得到什么」
- 从「了解实力」转向「查看案例」
- 从「机构介绍」转向「开设方向」

### 3. 简洁现代
- 清新蓝色渐变代替深色沉重
- 统一圆角系统
- 精简文案表达

### 4. 信任建立
- 真实数据（92.3%、1:8）
- 可验证案例
- 具体承诺（365天全年答疑）

## 🔄 如何复现修改

### 方法一：直接修改数据文件（推荐）

1. **修改首页数据**
   ```bash
   # 编辑文件
   vim cfqh/database/local-admin-data.json

   # 或使用 3200 端口管理后台
   npm run admin:web
   # 访问 http://localhost:3200
   # 进入「首页配置」进行修改
   ```

2. **修改首页代码**
   ```bash
   # 编辑文件
   vim cfqh/src/pages/home/index.tsx

   # 搜索并替换：
   # 1. Hero 背景渐变
   # 2. 圆角数值
   ```

3. **重启开发服务器**
   ```bash
   npm run dev:weapp
   ```
