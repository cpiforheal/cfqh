# 3200 后台 React 剩余迁移收尾摘要

## 本轮完成了什么

这轮把 React 后台里最后一个还没正式迁完的内容模块 `题库` 接入完成，同时把几张偏配置型主控区从重型 `ProTable` 收成更轻的 `Table + 抽屉` 结构，保证后台继续保持老师友好的 row 表格风格，但少带一层不必要的 Pro 依赖。

## 本轮迁移完成的模块

### 题库

文件：

- `admin-web-react/src/pages/QuestionBankPage.tsx`
- `admin-web-react/src/pages/question-bank/types.ts`
- `admin-web-react/src/api.ts`

本轮落地内容：

- 新增 React 版题库主控区，不再走占位页
- 主控区拆成 4 张真实数据表：
  - 题库入口区块总表
  - 题目表
  - 模拟卷表
  - 导入记录表
- 题目、模拟卷、导入记录都支持二级抽屉编辑
- 接上现有导入接口：
  - CSV 预校验
  - CSV 正式导入
- 继续沿用现有后端契约，不改 `/api/page/*`、`/api/collection/*` 和导入接口路径

  - 评估 `账号管理` 是否也脱离 ProTable
  - 继续拆分 `Directions` / `Learners` 的表单与高级能力

## 这轮形成的复用结论

后续如果再迁新的内容模块，优先按这套方式落地：

1. 能不用 `ProTable` 的纯配置页，就用 `Table + Card + Drawer`
2. 主表只保留顺序、区块、摘要、状态、操作
3. 复杂字段全部下沉到二级抽屉
4. 只有真正需要搜索、筛选、分页、列管理的重列表，再保留 `ProTable`
