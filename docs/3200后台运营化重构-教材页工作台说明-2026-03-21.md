# 3200 后台运营化重构说明（教材页工作台）

## 本轮目标

这轮不是继续补字段，而是先把 `3200` 后台里最容易让非开发人员迷失的教材页工作区收成更像“老师工作台”的结构。

核心目标：

- 减少一眼看到的词条数量
- 让使用者知道“下一步该改什么”
- 保留前后端字段一致性，不额外制造 CMS 和前端错位
- 不推翻原有 CMS UI，只做工作流和文案层的运营化重构

## 本轮改动范围

本轮只重构了 `教材资料` 这一页在 `3200` 后台里的工作区，不包含首页、方向页、师资页、成果页等其他页面。

前端教材页本身的数据结构没有新增本轮额外字段；本轮主要改的是后台可读性和任务流。

## 主要改动

### 1. 教材页工作区改成 5 步任务流

原来教材页会直接把顶部、切换、主推区、资料货架、集合条目都分散列出来，非开发者容易一下看到太多词条，不知道先改哪一项。

现在改成了 5 个更明确的任务块：

1. `第 1 步 · 页面顶部`
2. `第 2 步 · 主推区标题`
3. `第 3 步 · 主推套系内容`
4. `第 4 步 · 资料货架`
5. `第 5 步 · 底部咨询条`

这样做的目的：

- 把“字段思维”改成“页面任务思维”
- 先让老师按页面阅读顺序改
- 不要求老师先理解 CMS 数据结构

### 2. 新增“推荐修改顺序”提示区

在教材页工作区顶部新增了一组简短引导卡，告诉使用者：

- 先改页面顶部
- 再改主推大卡
- 最后补货架和咨询

这样可以减少第一次进入后台时的心理负担，也能避免“看见很多区块但不知道从哪开始”。

### 3. 教材页主工作区去掉重复表格

原来工作区同时存在：

- 卡片导航
- 表格 row

这会让信息重复、视觉噪音过大。

现在教材页工作区只保留卡片式任务入口，不再重复显示那张区块表格。

保留原因：

- 卡片对非开发人员更容易理解
- 每张卡已经能表达“改哪里 / 会影响哪里 / 下一步去哪”

### 4. 区块标题改成更偏运营使用的话术

例如：

- `顶部标题区` 改成 `第 1 步 · 页面顶部`
- `主推套系包` 改成 `第 3 步 · 主推套系内容`
- `货架资料卡` 改成 `第 4 步 · 资料货架`

目的不是改名本身，而是让名字直接告诉使用者：

- 当前在页面的什么位置
- 这一步是不是优先项

### 5. 资料货架条目收成“标题区 + 资料卡来源”

原来：

- `资料货架区`
- `货架资料卡`

会像两个平行概念，非开发人员不一定能第一眼理解区别。

现在逻辑更清楚：

- `第 4 步 · 资料货架`
  - 先改货架标题和滑动提示
  - 需要改卡片内容时，再通过关联入口跳去 `货架资料卡`

这样能让“区块标题”和“具体内容来源”分开理解。

## 用户看到的变化

进入 `3200 -> 教材资料` 后：

- 不会再先看到一大堆平铺词条
- 会先看到“推荐修改顺序”
- 主工作区只剩 5 个任务块
- 每个任务块都更像“页面编辑任务”，而不是“字段集合”

## 仍然保留的能力

本轮没有删除这些能力：

- 资料页所有前端展示内容仍然可改
- 主推套系包和货架资料卡仍然可以单独维护
- 前端与 CMS 字段依旧对齐
- 关联内容依旧可以通过跳转入口进入

## 本轮还没做的事

这次只是先把教材页工作区收成更可用，还没有做下面这些更深层的运营化能力：

- 其他页面同样的任务流重构
- 不同角色的后台视图裁剪
- 更强的“前端预览对照”
- 更细的“只给老师看到必要字段”

## 下一步建议

如果这版方向正确，建议按这个顺序继续：

1. 把 `首页` 工作区也改成任务流
2. 把 `开设方向` 工作区改成任务流
3. 为 `编辑老师 / 发布老师 / 管理员` 做不同层级的后台显示

## 本轮结论

这次重构的重点不是“后台能不能改”，而是“老师能不能轻松知道该怎么改”。

教材页这条链路现在已经从偏开发者的 CMS 操作方式，向更偏运营老师可理解的工作台迈了一步。
