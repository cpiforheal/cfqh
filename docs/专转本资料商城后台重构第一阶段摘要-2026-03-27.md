# 专转本资料商城后台重构第一阶段摘要

日期：2026-03-27

## 1. 本轮目标

本轮先做“后台数据逻辑迁移 + React 主控区接入”，不强行改动小程序商城前端 UI。

原则：

- 小程序前端展示尽量保持不变
- 3200 后台仍保持老师友好的表格主控区
- 先把商城底层从“页面配置型资料区”升级成“资产 + 商品 + 内容项 + 权益”模型
- React 后台只展示老师真正需要维护的关键条目，复杂字段收进二级抽屉

## 2. 新的业务对象

本轮已经把商城主干切成 4 层：

1. `mallAssets`
   - 资料资产
   - 存 PDF、封面、页数、文件大小、预览页数、状态
2. `mallProducts`
   - 商城商品
   - 存商品标题、简介、价格、免费/付费、上架状态
3. `mallProductItems`
   - 商品内容项
   - 维护商品和资料资产之间的关联关系
4. `mallEntitlements`
   - 用户权益
   - 记录用户领取/购买后获得的实际内容使用权

## 3. 兼容策略

为了不改当前小程序商城 UI，本轮没有直接让前端商城页读取新结构，而是：

- 后台存储逐步迁到 `mallAssets / mallProducts / mallProductItems / mallEntitlements`
- 服务端继续向小程序提供兼容后的旧商城货架结构
- 如果新集合为空，服务端会自动从旧 `materialPackages / materialItems` 生成兼容数据
- 如果新集合已有数据，服务端会优先把新结构映射成旧前台可消费的数据

这样可以做到：

- 老师后台先升级
- 小程序前端先不动
- 新旧数据结构在迁移期可以并存

## 4. 3200 React 后台本轮改动

### 4.1 商城主控区

商城主控区改成了 4 张表格卡片：

1. 商城关键区块
   - 维护顶部信息、阶段按钮、主推商品区、内容列表区、底部咨询条
2. 当前阶段商品
   - 维护当前学科与阶段下的商品
3. 当前商品内容项
   - 维护商品下挂了哪些内容卡片
4. 资料资产库
   - 维护 PDF、封面、预览页数等资产信息
5. 最近权益记录
   - 查看最近的领取/购买权益状态

### 4.2 抽屉表单

复杂字段全部收进抽屉，不在主表摊开：

- `MediaSectionEditorDrawer`
  - 改页面区块文案
- `MediaPackageEditorDrawer`
  - 改商品
- `MediaItemEditorDrawer`
  - 改商品内容项
- `MediaAssetEditorDrawer`
  - 改资料资产

老师在主表看到的是：

- 现在前台哪一块
- 当前显示的摘要
- 点进去会改什么
- 状态是否完整

## 5. 为什么这样改

这轮不是做“更炫的商城后台”，而是做“老师第一次进来也能理解的商城后台”。

主控区设计原则：

- 表格行必须对应前台真实展示位置
- 主表只保留高密度摘要
- 详细字段统一放进二级抽屉
- 优先按老师操作顺序组织，不按数据库结构裸露

老师的理解顺序应该是：

1. 先把资料录进资料资产库
2. 再创建商品
3. 再给商品挂内容项
4. 最后让小程序继续按原 UI 展示
