## 二、「类苹果」设计核心特征

你想要的设计语言，核心是以下几个点：

### 2.1 毛玻璃 (Glassmorphism)

**苹果的设计特征：**
- 背景模糊 + 半透明填充 + 细微边框
- 多层叠加时产生深度感
- 背景图像/颜色渗透产生「虚化透光」效果

**当前差距：**
```css
/* 你的当前实现 */
background: linear-gradient(180deg, var(--panel-bg-strong) 0%, var(--panel-bg) 100%);
backdrop-filter: blur(22px);

/* 苹果风格参考 */
background: rgba(255, 255, 255, 0.72);
backdrop-filter: blur(28px) saturate(180%);
border: 0.5px solid rgba(255, 255, 255, 0.3);
box-shadow: 0 8px 32px rgba(0, 0, 0, 0.08);
```

**改进建议：**

```css
/* 新增：强化毛玻璃类 */
.glass-panel {
  background: rgba(255, 255, 255, 0.72);
  backdrop-filter: blur(28px) saturate(180%);
  -webkit-backdrop-filter: blur(28px) saturate(180%);
  border: 0.5px solid rgba(255, 255, 255, 0.5);
  box-shadow:
    0 4px 16px rgba(0, 0, 0, 0.04),
    0 1px 4px rgba(0, 0, 0, 0.02);
}

/* 暗色模式 */
:root[data-theme='dark'] .glass-panel {
  background: rgba(30, 30, 30, 0.72);
  border-color: rgba(255, 255, 255, 0.1);
}

/* 悬浮态：更透明、更模糊 */
.glass-panel:hover {
  background: rgba(255, 255, 255, 0.82);
  box-shadow:
    0 8px 24px rgba(0, 0, 0, 0.06),
    0 2px 8px rgba(0, 0, 0, 0.04);
}
```

### 2.2 懒加载与骨架屏 (Skeleton Loading)

**苹果的特征：**
- 内容加载时显示灰色脉冲骨架
- 骨架有微妙的渐变 shimmer 动画
- 加载完成后内容「淡入」而非「跳入」

**当前状态：**
你已有 `boot-splash-skeleton`，但仅用于启动页。

**改进建议：**

```css
/* 骨架屏基础 */
.skeleton {
  background: linear-gradient(
    90deg,
    rgba(230, 230, 230, 0.8) 0%,
    rgba(250, 250, 250, 1) 50%,
    rgba(230, 230, 230, 0.8) 100%
  );
  background-size: 200% 100%;
  animation: shimmer 1.5s ease-in-out infinite;
  border-radius: 8px;
}

/* 卡片骨架 */
.skeleton-card {
  height: 180px;
  border-radius: 16px;
  animation: pulse 2s ease-in-out infinite;
}

@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}

@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.6; }
}

/* 淡入动画 */
.fade-in {
  animation: fadeIn 0.4s ease-out forwards;
}

@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(8px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
```

**需要 JS 配合：**
- 列表渲染使用虚拟滚动（超过 50 条）
- 图片使用 Intersection Observer 懒加载
- 内容区域进入视口时触发加载

### 2.3 拖拽交互 (Drag & Drop)

**苹果的特征：**
- 卡片可拖拽排序
- 拖拽时有「悬浮感」和「投影」
- 放下时有弹性动画

**当前差距：** 你的系统完全无拖拽功能。

**改进建议：**

```css
/* 拖拽卡片 */
.draggable {
  cursor: grab;
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}

.draggable:active {
  cursor: grabbing;
}

/* 拖拽中 */
.dragging {
  opacity: 0.9;
  transform: scale(1.02) rotate(1deg);
  box-shadow: 0 20px 40px rgba(0, 0, 0, 0.15);
  z-index: 100;
}

/* 放置位置指示 */
.drag-over {
  border: 2px dashed var(--accent);
  background: rgba(1, 193, 217, 0.05);
}
```

**需要 JS 配合：**
- 使用 SortableJS 或原生 HTML5 Drag API
- 侧边栏导航支持拖拽排序
- 卡片列表支持拖拽调整顺序

## 四、CSS 变量调整建议

在 `:root` 中调整以下变量，可快速获得更「苹果化」的效果：

```css
:root {
  /* 面板：更透明、更模糊 */
  --panel-bg: rgba(255, 255, 255, 0.64);
  --panel-bg-strong: rgba(255, 255, 255, 0.8);

  /* 阴影：更柔和 */
  --float-shadow: 0 8px 24px rgba(0, 0, 0, 0.06);
  --float-shadow-strong: 0 12px 32px rgba(0, 0, 0, 0.08);

  /* 圆角：更大 */
  --panel-radius: 20px;
  --card-radius: 16px;

  /* 边框：更细更淡 */
  --line-faint: rgba(0, 0, 0, 0.06);
}

:root[data-theme='dark'] {
  --panel-bg: rgba(30, 30, 30, 0.64);
  --panel-bg-strong: rgba(40, 40, 40, 0.8);
  --line-faint: rgba(255, 255, 255, 0.08);
}
```

## 五、你真正想要的设计参考

如果你想要更彻底的「苹果化」，建议参考以下模式：

### 5.1 登录页风格（已部分实现）

你现在登录页的 `boot-splash-panel` 已经接近苹果风格：
- 圆角 28px
- blur(20px) + 白色半透明
- 内阴影高光

**可继续强化：**
- 增加「噪声纹理」背景
- 添加微妙的「光泽」动画

### 5.2 卡片设计

**当前：** 扁平 + 边框 + 阴影
**苹果化目标：** 毛玻璃 + 悬浮升压 + 弹性动画

```css
/* 推荐卡片样式 */
.card-apple {
  background: rgba(255, 255, 255, 0.7);
  backdrop-filter: blur(24px) saturate(160%);
  border: 0.5px solid rgba(255, 255, 255, 0.4);
  border-radius: 18px;
  transition: all 0.25s cubic-bezier(0.25, 0.46, 0.45, 0.94);
}

.card-apple:hover {
  transform: translateY(-4px) scale(1.01);
  background: rgba(255, 255, 255, 0.85);
  box-shadow: 0 20px 40px rgba(0, 0, 0, 0.08);
}
```

### 5.3 侧边栏导航

**当前：** 固定宽度 + 收缩动画
**苹果化目标：** 毛玻璃 + 图标 + 标签组合

```css
.nav-item {
  display: flex;
  align-items: center;
  gap: 12px;
  padding: 10px 14px;
  border-radius: 12px;
  transition: all 0.2s ease;
}

.nav-item:hover {
  background: rgba(0, 0, 0, 0.04);
}

.nav-item.active {
  background: rgba(0, 0, 0, 0.06);
  color: #007AFF; /* 苹果蓝 */
}
```

## 六、结论与建议

### 6.1 你现在缺的是什么？

| 你有的 | 你缺的 |
|-------|--------|
| 基础毛玻璃 | 多层次、饱和度更高的 blur |
| 启动骨架屏 | 业务页内容骨架屏 |
| 简单 hover | 弹性、缩放、阴影组合动画 |
| 静态列表 | 可拖拽排序 + 拖拽视觉反馈 |

### 6.2 推荐实施路径

**第一阶段（1-2天）：CSS 变量调整**
- 修改 `:root` 中的面板、阴影、圆角变量
- 新增 `.glass-panel` 类统一使用
- 统一过渡动画曲线

**第二阶段（2-3天）：交互增强**
- 按钮/卡片 hover 效果
- 骨架屏组件化
- 页面切换过渡

**第三阶段（3-5天）：拖拽功能**
- 引入 SortableJS
- 侧边栏/列表拖拽排序
- 拖拽视觉反馈
