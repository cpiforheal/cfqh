# 开设方向页面 UI 审查与改版建议

日期：2026-03-19  
范围：小程序「开设方向」页面  
审查目标：从 `首次了解专转本培训的新学员` 与 `产品转化率` 两个角度，分析当前页面风格、存在的问题与改版建议。

## 一句话结论

当前「开设方向」页已经具备基础信息完整度，但整体更像“方向信息陈列页”，还不是“帮助新学员快速判断自己适合哪条线，并顺势进入咨询”的高转化页面。  
最大问题不在于信息少，而在于 `决策顺序不清晰`、`视觉语言不统一`、`咨询动作承接太弱`。

## 当前页面的整体风格

### 现状判断

- 页面延续了全站偏明亮、卡片化的基础风格，整体不压抑，阅读压力不算大。
- 顶部标题区比较克制，但也因此第一眼冲击力不足，无法快速建立“这是帮助我选方向的页面”这一认知。
- 页面内部混用了浅灰、偏紫、深色卡片、浅色卡片几套语言，风格不够统一。
- 方向卡本身信息量不少，但排布方式更像“介绍课程内容”，不够像“帮学生判断自己该选哪一个”。

### 风格优点

- 明亮、干净，至少不会产生廉价教培页的压迫感。
- 卡片分层清楚，基础可读性不错。
- 两个核心方向已经有明显主次，不是完全平铺。

### 风格不足

- 与首页当前蓝白、积极、可信的风格衔接不够，方向页里偏紫和深色块会让页面气质突然变重。
- 顶部没有形成“强判断入口”，学生进入页面后仍要自己消化信息。
- 视觉层级更像“模块堆叠”，不是“转化路径引导”。

## 主要问题与优先级

## P1 高优先级

### 1. 页面第一屏没有快速回答“我该看哪一个方向”

当前顶部只有 `标题 + 说明 + 分类条 + 建议卡`，但没有形成明确的判断入口。  
学生进入页面时最需要的是一句很直白的话：`你是医护背景，还是高数薄弱/跨考？`  
现在的页面需要先读标题、再读副标题、再看建议卡，决策成本偏高。

为什么优先级高：
- 这是方向页的核心任务。
- 第一屏如果不能帮助判断，用户会把页面当作“普通介绍页”快速滑过。

建议：
- 将第一屏改成“方向判断首屏”，而不是“页面标题首屏”。
- 用一句主标题 + 一句副标题，直接服务判断，例如：
  - 主标题：`先判断适合哪条备考线`
  - 副标题：`医护类同学优先看医护大类，理工/经管且数学薄弱优先看高数专项`
- 建议卡不要只做说明，应直接变成强引导模块，提供两条明确路径。

### 2. 分类条看起来可操作，但当前价值感弱，容易造成“伪交互”

顶部分类条目前更像装饰性筛选器，视觉上像可切换标签，但页面核心内容并没有围绕它形成强反馈。  
这会让用户误以为自己可以快速筛选，结果滑下去看到的仍然是固定内容。

为什么优先级高：
- 它占据首屏高注意力区域。
- 伪交互会削弱页面专业感和可信度。

建议：
- 二选一。
- 要么把分类条做成真正有明显切换反馈的筛选条。
- 要么弱化它的存在感，改成更直接的“你属于哪类同学”入口。
- 如果短期不做真实筛选，建议不要把它做得太像可操作组件。

### 3. 方向卡信息很多，但“决策信息”和“课程介绍信息”混在一起

当前方向卡中有：
- 标签
- 标题
- 适合学员
- 培养方式与内容
- chips

这套结构信息不算少，但学生真正最先关心的是：
- 我适不适合这张卡
- 这条线解决我什么问题
- 我下一步该做什么

而当前方向卡更偏“课程信息说明”，不是“决策卡”。

为什么优先级高：
- 方向卡是本页最重要的转化模块。
- 如果卡片不能帮用户快速判断，页面再美也很难提高咨询率。

建议：
- 方向卡应改成“判断优先”的结构。
- 推荐顺序：
  1. 方向名称
  2. 一句话适合谁
  3. 这一方向能解决什么问题
  4. 2-3 个核心支持点
  5. 明确动作按钮，如“看课程安排”或“先做学情评估”
- “培养方式与内容”这一标题太偏机构视角，可改成更学生化的表达，如“你会重点学到什么”。

### 4. 页面缺少清晰的咨询承接区，转化链在方向卡之后断掉了

目前页面底部只有“更多专业方向”模块，没有一个真正承接咨询的动作区。  
这意味着用户即使看完两张方向卡，也缺少一个自然的下一步。

为什么优先级高：
- 转化页没有承接动作，就会变成浏览页。
- 用户看完后要自己决定去哪里，流失率会很高。

建议：
- 在两个方向卡之后增加一个简洁但可信的咨询承接区。
- 核心文案应围绕：
  - 不确定选哪个方向
  - 想确认课程安排
  - 想先做学情评估
- CTA 不要写得太销售，建议更像“先确认适合的备考线”。

### 5. 页面存在开发者/系统视角信息，不适合 C 端展示

当前页面仍挂着 [CmsSyncBadge](/Users/cpiforheal/Documents/cfgh/cfqh/src/pages/courses/index.tsx) 以及云端加载失败提示。  
这类信息对运营或开发有用，但对学生没有价值，还会破坏沉浸感。

为什么优先级高：
- 会直接伤害专业感。
- 会让学生感知到“这是个半后台状态的页面”。

建议：
- C 端页面默认不展示 CMS 同步状态与技术错误提示。
- 若确需保留，应仅在后台预览模式中可见。

## P2 中优先级

### 6. 页面色彩与首页不够统一，影响品牌连续性

首页当前已经逐步转向明亮、可信、偏蓝白的语言，但方向页里：
- 建议卡偏紫
- 部分方向卡使用深色块
- 辅助标签和按钮也有多套颜色来源

整体会让用户感觉像进入了另一套界面。

建议：
- 方向页主色统一到首页当前的清爽可信体系。
- 保留不同方向的区分，但不建议使用“一个深色、一个浅色”的强跳变方案。
- 更适合做成“同品牌体系下的两种方向语义色”。

### 7. 建议卡有帮助，但目前更像说明文本，不像高价值入口

“不知道如何规划专业方向？”这张卡片本质上承担的是高价值入口，但现在视觉和内容还偏软。  
它更像一段说明，而不是一个“我应该点这里”的动作区。

建议：
- 提升这张卡的权重，但不要比方向卡更重。
- 把 3 条建议压缩成更利于扫读的短句。
- CTA 从“免费获取 1对1 专业规划”改为更自然、更低压力的表达，如“先做方向判断”。

### 8. 页面缺少足够的“可信证明”

当前方向页有课程信息，但缺少能增强信任的佐证，例如：
- 这个方向适合哪些同学
- 是否有阶段测评
- 是否有答疑 / 督学
- 是否已有成熟经验

页面不是完全没有说这些，但表达比较分散，没有形成“为什么可以信”的模块。

建议：
- 不一定要加很多内容，但至少应在每张方向卡里明确 1 个“可信点”。
- 比如：
  - 医护大类：专业课与公共课同步推进
  - 高数专项：基础薄弱也有分阶段训练

## P3 低优先级

### 9. “更多专业方向”模块当前更像打断，不像加分

“筹备中”的模块在页面底部会削弱已建立起来的主方向认知。  
用户刚刚看完两个重点方向，底部再出现“更多方向筹备中”，容易把注意力带向“不完整感”。

建议：
- 弱化这个模块的视觉权重。
- 它更适合作为补充说明，而不是底部一个完整卡块。

### 10. 页面缺少更明确的视觉节奏

目前结构是：
- 标题
- 分类
- 建议卡
- 两张方向卡
- 更多方向

信息是完整的，但节奏偏平均。每个模块体量比较接近，导致页面缺少“主高潮”。

建议：
- 建立更明确的浏览节奏：
  1. 第一屏判断
  2. 两个核心方向二选一
  3. 咨询承接
  4. 补充说明

## 建议中的改版方向

## 方向一：保持当前 UI 基础，做“轻重构”

适合：你希望保留现有结构，不大动页面骨架。

建议做法：
- 顶部改成更强的判断型首屏
- 分类条弱化或做真筛选
- 建议卡改成“方向判断入口”
- 两张方向卡改成更明确的决策卡
- 增加底部咨询承接区
- “更多专业方向”降权

预期收益：
- 不需要大改 UI 架构
- 成本相对低
- 能明显提升首次访问用户的理解效率和咨询动机

## 方向二：把页面改成“二选一决策页”

适合：你希望这页承担更强的转化任务。

建议做法：
- 首屏直接告诉用户“先选方向”
- 两个核心方向卡作为页面绝对主角
- 每张卡片都包含：
  - 适合谁
  - 主要问题
  - 重点支持
  - 下一步动作
- 底部只保留一个轻量补充区和咨询区

预期收益：
- 页面目标更单一
- 更适合新学员第一次进入时快速做决定
- 与首页“先判断方向，再安排课程”的策略更一致

## 推荐本轮修改顺序

1. 先重构顶部首屏表达，让用户 3 秒内知道自己该看什么。  
2. 再调整两张方向卡的信息结构，让“适合谁”成为第一重点。  
3. 增加底部咨询承接区，把页面从介绍页变成转化页。  
4. 最后再统一配色、阴影、模块权重，解决风格一致性问题。  

## 最终判断

从学生视角看，这个页面目前“能看懂”，但还不够“能帮助我快速下决定”。  
从产品视角看，这个页面目前“有内容”，但还不够“有转化路径”。  

最值得改的，不是继续加信息，而是把已有信息重新排成一条更清晰的路径：

`先判断适合方向 -> 再理解这个方向怎么带你学 -> 最后顺势进入咨询`
